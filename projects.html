Paxos Chatroom Programming partner and I implemented a "chatroom" with clients that send messages to each other through servers. The servers and clients exist in an asynchronous system so the servers decide the ordering of messages based on a Paxos protocol. CS378H: Computer Vision and Machine Learning Honors, Fall 2015

Sliding Window Face Detector Given an image, this program will detect the faces present in it. The system was first trained on both positive features (6,713 images of faces) and negative features (20k images of things that were not faces). Every image in each features set was converted to a HoG map. The sets of HoG maps were used train a support vector machine (SVM). After converting an image to a HoG map, the program uses sliding windows at different scales, checking the content inside the window with the SVM. If a face seems to be detected, a vote will be casted for that window. After running the sliding window, the program will use non-maximum suppression to decide the best centers for each detected face.
Image Recognition Search This program creates a bag of words "visual vocabulary" by running a k-means algorithm over SIFT descriptors various frames of a video. The program has methods to search for frames that are visually similar to a query frame and to search for other occurences of an object within the rest of the video.
Image Stitching - Panoramas Using corresponding SIFT descriptors to calculate affine transformations, this program stitches together two images to form one continuous image. Examples of images I made can be found here and here.
Color Quantizer Given an image and a number, this program quantized the colors of an image based on either RGB or HSV values. Quantization was done using a k-means algorithm.
Hough Space Circle Detector I implemented a program that detected circles of a given pixel radius in a given image using a Hough transform voting space.
Seam Carving Content-Aware Image Shrinker I implemented the image-shrinking algorithm described in "Seam Carving for Content-Aware Image Resizing" by Avidan and Shamir.

Heap I implemented a heap that allowed for memory allocation (malloc) and freeing. My heap used a linked list to track available block, was able to merge freed blocks in both directions, and followed the first-available protocol.
Preemptive Multithreading I implemented preemptive multithreadings, where threads were interrupted after a certain number of clock ticks and forced to yield to other threads. Each thread was created using heap memory so a reaper thread existed to free threads that had exited.
Cooperative Multithreading I implemented cooperative multithreading, where threads allow other threads to make progress by yielding to each other. Each thread had its own stack space. When a thread yielded, I made sure its registers were saved so they could be restored when the thread became active again.

CS314H: Data Structures and Algorithms Honors, Fall 2014

Webcrawler, Database, and Web Query This project consisted of multiple parts that led to being a search engine applet. The first part was a webcrawler. Given a URL from a dummy internet, the webcrawler will parse through the webpage's HTML, saving the page's content to a database and searching for links. The found links will be add to a queue, and the webcrawler use the queue to find pages it has yet to visit. The database was an inverted index where each word found in the internet was a key and its value was a list of its found locations (URL and index into page). To query the database, a user can type queries into the search engine applet using a query language. A query parser seperated the query into keywords and symbols and then queried the database accordingly, returning relevant web results.
Treap My programming partner and I implemented treaps, a data structure that combined properties of a binary search tree with a binary heap. Each treap node has a value. which determines the node's placement if the treap was a BST, and a randomly generated id, which determines the node's placement if the treap was a heap. The treap also allowed for a user to add or remove nodes.
